我以前一直琢磨着，什么时候学学 Flutter。

## 上次

很久以前上一次想尝试，然后还是以 **RN** 开发上架 AppStore 一款普通工具，收尾。

当时，我觉得我的需求不难，也只是想实践一下 App 开发，开发出能上架使用的 **App/Apk** 即可。Flutter 不是必选项。

## 这次

Flutter 时不时能看到更新、看到一些讨论，我又生起用 **Flutter** 做 AI Chat 学习一下。或者转而做 Windows 工具试试，之前用 Python + pywebview 做 App，转而试试 Python + Flutter。

研究研究着，发现市面上更常用的方案，是 Electron 方案，有的更杂，Electron + Flutter + C++ DLL 等等。

![](/blogs/plan-b/42f85208ac97023f.png)

## 质疑

又开始升起质疑，要不要试试 Flutter 呢？

第一个答案，是，因为 Flutter 可以拓展**知识面**，拓展移动端更好。

但另一个更实在的叮咛是：

我这个年份经验的程序员，更需要面临的问题是

——“**是否能够解决更复杂的问题？**”

——“**是否独立开发一款产品？**”

独立的工程能力，大于学习某一个单一能力。我需要的是，完善产品的**工程链**，而不是 RN 再补充 Flutter。

![](/blogs/plan-b/e6f462dc22b6bc6f.webp)

## 放下

对于 Flutter，这次的答案，依旧是放下。

我到不是对 Flutter 执着，或者认为 Flutter 比 RN 更好。我只是**潜意识**想起，我是不是要学习，完善大前端之类的方向。

我更在乎的是，做出一个大家认为**好用的产品**。

## Flutter 

或许以后，因为工作需要，我会好好去学学，像一次知识旅行。

现在，Electron 工程化产品，更有价值。
